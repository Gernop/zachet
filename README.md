# ИС «Пассажирское автопредприятие»

Полноценное веб-приложение с REST API бэкендом на Express.js и SQLite базой данных.

## 📋 Требования

- Node.js 20.19+ или 22.12+
- npm

## 🚀 Запуск приложения

### Шаг 1: Установка зависимостей

```bash
npm install
```

### Шаг 2: Запуск сервера (бэкенд)

```bash
npm run server
```

Сервер запустится на **http://localhost:3000**

API доступно на **http://localhost:3000/api**

### Шаг 3: Запуск фронтенда (в новом терминале)

```bash
npm run dev
```

Фронтенд откроется на **http://localhost:5173**

## 🔐 Учётные данные

### 👨‍💼 Сотрудники (доступ к админ-панели)

| Логин | Пароль | Роль | Возможности |
|-------|--------|------|-------------|
| admin | admin123 | **Администратор** | Полный доступ + управление пользователями |
| dispatcher | disp123 | **Диспетчер** | Управление автобусами, водителями, маршрутами, рейсами |
| manager | mgr123 | **Менеджер** | Просмотр и управление данными |

### 🧑 Пассажиры (только просмотр и бронирование)

| Роль | Возможности |
|------|-------------|
| **Пассажир** | Просмотр маршрутов, бронирование поездок, просмотр своих бронирований |

> Для создания пассажи зайдите как `admin` → Админ-панель → Пользователи → Добавить пользователя → Роль: "Пассажир"

## 📡 API Эндпоинты

### Авторизация
- `POST /api/auth/login` — вход в систему

### Пользователи
- `GET /api/users` — получить всех пользователей
- `POST /api/users` — создать пользователя
- `PUT /api/users/:id` — обновить пользователя
- `DELETE /api/users/:id` — удалить пользователя

### Бронирования
- `GET /api/bookings` — получить все бронирования
- `POST /api/bookings` — создать бронирование
- `PUT /api/bookings/:id` — обновить бронирование
- `DELETE /api/bookings/:id` — удалить бронирование

### Тарифы
- `GET /api/tariffs` — получить все тарифы
- `POST /api/tariffs` — создать тариф
- `PUT /api/tariffs/:id` — обновить тариф
- `DELETE /api/tariffs/:id` — удалить тариф

### Автобусы
- `GET /api/buses` — получить все автобусы
- `POST /api/buses` — создать автобус
- `PUT /api/buses/:id` — обновить автобус
- `DELETE /api/buses/:id` — удалить автобус

### Маршруты
- `GET /api/routes` — получить все маршруты
- `POST /api/routes` — создать маршрут
- `PUT /api/routes/:id` — обновить маршрут
- `DELETE /api/routes/:id` — удалить маршрут

## 🛠 Технологии

**Фронтенд:**
- Vue 3 (Composition API)
- Vite
- CSS3

**Бэкенд:**
- Express.js
- SQLite (sql.js)
- bcryptjs (хеширование паролей)

**База данных:**
- Таблица `users` — пользователи системы
- Таблица `bookings` — бронирования пассажиров
- Таблица `tariffs` — тарифы на перевозки
- Таблица `buses` — автобусы предприятия
- Таблица `routes` — маршруты перевозок

## 🔒 Безопасность

- Пароли хешируются с помощью bcryptjs
- CORS настроен для localhost:5173
- SQL-запросы выполняются через параметризованные запросы SQLite

## 💾 База данных

SQLite база данных создаётся автоматически при первом запуске сервера в папке `data/`.
Содержит таблицы: `users`, `bookings`, `tariffs`, `buses`.
